Important Concepts - HTML/CSS

Lesson1: The Basics of the Web and HTML

How the web Works

Computers communicate with each other over the internet. HTTP Request to server. Server responds with document or image to the web browser of the user's pc. For example, this link demostrates this concept.

HTML

HTML stands for Hypertext Markup Language.HTML documents form the majority of the content on the web. HTML documents contain text content which decribes "what you see" and markup which decribes "how it looks".

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags.

Why Computers are Stupid

Computers interpret instructions literally. A small programming mistake can cause huge problems in the program.

Inline vs. Block Elements

HTML element are either inline or block. Block elements form an invisible box around their content.

Lesson 2: Creating a Structured Document with HTML

Developer Tools (in the browser)

HTML elements are either inline or block.

The tree-like structure of HTML

The tree-like structure, like a family-tree with the elements acting as mother and children. In a browser, this structure shows up as a series of nested boxes.

The relationship between indented HTML and boxes

WHen you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

Text Editors (for Coding)

When coding, special text editors are used.

Lesson 3: Adding CSS Style to HTML

What is CSS

CSS has rules. The most specific rules applies to every element. CSS uses a computer science principle called inheritance.

AVOID REPETITION! - Use Selectors!

There several types of selectors. BASIC, COMBINATORS, PSEUDO, to name a few. Using selectors and classes helps organize and sub-group your code so that it may be reused in a efficient way. Avoiding repetition saves time and could save computing resources. For more info please see documentation on the mozilla developer network site.

Utilize the Box Model

A web page is made up of boxes. Per the box model: Content, Padding, Border, Margin. Keep in mind how different browsers will render your code.

Code, Test, Refine

Code, boxify, position. Look for natural boxes. Look for repeated styles and semantic elements. Code HTML. Apply styles from the biggest to smallest. Then fix.
